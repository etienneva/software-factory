# Workflow Standard

Tout projet suit obligatoirement le cycle suivant :

## Étape 1 : Project Brief

Description du problème à résoudre.

## Étape 2 : Product Design

Création :

* product.md
* MVP
* user stories
* roadmap

## Étape 3 : Architecture

Création :

* architecture.md
* modèle de données
* choix techniques

## Étape 4 : Backlog

Découpage en :

* Epics
* Features
* Tasks

## Étape 5 : Développement

Chaque tâche :

* une branche Git
* une Pull Request

## Étape 6 : QA

Validation :

* critères d'acceptation
* tests
* sécurité

## Étape 7 : Déploiement

* staging
* production

## Étape 8 : Monitoring

* logs
* analytics
* incidents
