# Testing Standards

## Objectif

Garantir la stabilité du produit.

## Couverture minimale

### Tests unitaires

Logique métier critique.

### Tests fonctionnels

Parcours utilisateur principaux.

### Tests de régression

Fonctionnalités déjà livrées.

## Cas obligatoires

* succès
* erreur
* utilisateur non connecté
* permissions insuffisantes
* données invalides

## Outils recommandés

Frontend :

* Vitest
* Playwright

Backend :

* Jest
* Tests API
