# Coding Standards

## Général

Le code doit être :

* lisible
* documenté
* maintenable

## Règles

### Nommage

Variables :

camelCase

Fonctions :

camelCase

Composants :

PascalCase

### Fichiers

Un fichier = une responsabilité.

### Fonctions

Favoriser :

* petites fonctions
* responsabilité unique

### Commentaires

Commenter uniquement :

* logique complexe
* arbitrages importants

Ne jamais commenter l'évident.

### Duplication

Éviter toute duplication importante.

Refactoriser dès le deuxième cas significatif.
