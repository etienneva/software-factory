# Definition of Done

Une fonctionnalité n'est considérée comme terminée que si les critères suivants sont remplis.

## Produit

* Les critères d'acceptation sont validés.
* Le Product Agent confirme que le besoin est couvert.
* Les cas d'usage principaux fonctionnent.

## Code

* Le code compile sans erreur.
* Le code respecte les standards du projet.
* Aucun TODO bloquant n'est laissé.

## Tests

* Les tests unitaires passent.
* Les tests fonctionnels passent.
* Les scénarios critiques ont été validés.

## Documentation

* La documentation est mise à jour.
* Les décisions importantes sont documentées.

## Sécurité

* Les permissions ont été vérifiées.
* Aucun secret n'est exposé.

## Déploiement

* La fonctionnalité est déployable.
* Aucun impact majeur sur l'existant n'est détecté.

Une Pull Request ne peut être fusionnée que si tous les critères ci-dessus sont respectés.
